# Editorial Redesign — keepcalmandcarryon.help

**Date**: 2026-03-03
**Status**: Approved
**Approach**: B — rebuild presentation layer from the working API/data layer up

## Context

The site serves UAE residents and stranded travellers during active Middle East conflict. The backend (ingestion pipeline, Supabase schema, briefing engine, flight query API, chat endpoint) is solid. The presentation layer needs a full redesign to serve anxious users rather than ops professionals.

## Primary Users

1. **Stranded traveller** — stuck overseas, trying to get back to Dubai/Abu Dhabi. Needs: Is airspace open? Are flights running? What's my best route?
2. **UAE resident monitoring** — living in Dubai/Abu Dhabi, checking safety. Needs: Should I change plans? Is DXB operating? Any new advisories?

## Design Principles

- **Status first, details second** — above-the-fold answers "what do I need to know right now?"
- **One query interface** — single input that auto-routes between flight lookups and GPT chat
- **Mobile-first** — designed for 375px, enhanced for desktop
- **Bold modern editorial** — BBC News app meets Apple Weather. Authoritative and digital-native.
- **Facts, not feelings** — no speculation, no invented content. Every statement traceable to an official source.

### Motion

- CSS transitions: 200-300ms ease-out for state changes
- Status indicator: slow opacity pulse when disruptions active
- Section entry: translateY(8px) + opacity fade on scroll
- Query results: height expansion + opacity fade
- No bouncy/playful animations

### Layout

- Full-width status hero (edge-to-edge on mobile)
- Content: max-width 720px for editorial sections, 1100px for data sections
- No sidebar on any viewport
- Generous vertical rhythm between sections
- Cards with subtle shadow (`shadow-sm`) on white backgrounds

## Information Architecture

### 1. Status Hero (above the fold)

The hero adapts based on three states from `deriveAirspacePosture()`:

**Normal/Open**:
- Background: `--surface-dark`
- "UAE Airspace Open" with green dot, gentle pulse
- One-line subtitle about operations
- Inline metrics: tracked / delayed / cancelled

**Heightened/Disruptions**:
- Background: gradient `--surface-dark` to `#7C2D12` (orange-900)
- "Disruptions Reported" with amber pulsing dot
- Specific disruption info from briefing
- Metrics colour-coded amber/red

**Unclear**:
- Background: `--surface-dark` to `#1E293B`
- "Status Unclear — Data Limited" with gray dot
- "Verify official channels directly"

**Unified query input** inside the hero:
- White bg, rounded-xl, shadow-lg
- Placeholder: "Ask anything — flights, routes, safety, advisories..."
- Scenario chips below: dynamic from `buildFlightPromptSuggestions()` + hardcoded defaults
- Results render inline below chips, still on dark background
- Auto-routes: flight intent -> `/api/flights/query`, general -> `/api/chat`

**Sticky behaviour**: When user scrolls past hero, a compact header appears with site name + "Ask" button that scrolls back to query input.

### 2. Situation Briefing

White card on stone background. Replaces "Current State Briefing."

- Label: "SITUATION BRIEFING" — 11px, tracked uppercase, text-secondary
- Briefing paragraph from extractive pipeline, DM Sans 16px/1.65
- Amber divider if confidence is "medium" or "low"
- Footer: relative update time, confidence level, source count

Content generated by existing `buildFallbackBriefParagraph()` — no changes to the pipeline.

### 3. Flight Pulse

Two airport cards (DXB, AUH) side-by-side on desktop, stacked on mobile.

Each card:
- Airport code in DM Serif Display 24px
- Stat lines: Tracked / Delayed / Cancelled with colour-coded numbers
- Latest fetch timestamp
- Expandable top routes list

### 4. Latest Updates (Live Ticker)

Vertical editorial feed replacing current `LiveUpdatesTicker`.

Each update card:
- Coloured status dot (left)
- Source name (bold) + relative timestamp (right)
- Headline: DM Sans 15px semibold
- Summary: 14px, 2-line clamp
- Footer: source type badge, validation badge, "Open original" link
- Tap to expand: full text + provider history inline (no side panel)

Filter row: "All" / "Advisories Only" / "Airlines" / "Government" toggle pills
Live indicator: "Live" badge with slow pulse dot

### 5. My Tracking

Functionally identical to current. Visual updates:
- Status badges using new colour system
- Cloud sync/auth in collapsed "Settings" subsection

### 6. Resources & Contacts

**Not collapsed.** Three subsections:

1. **Emergency & Wellbeing** — 800 HOPE prominently displayed, calm green accent
2. **Airlines** — Emirates, Etihad, flydubai, Air Arabia, Oman Air, Qatar Airways. Phone (`tel:` links), website, X handle.
3. **Government & Embassies** — UAE MOFA, GCAA, GDRFA, India MEA, UK FCDO, Australian DFAT

### 7. Source Health (footer)

Minimal footer strip:
- "X of Y sources reporting normally · Z temporarily unavailable"
- Expandable list of unavailable sources with reason and direct link

## Mobile-Specific Requirements

- No floating FAB/pill — query input is in the hero
- Sticky compact header appears on scroll past hero
- All touch targets minimum 44px
- Font sizes: minimum 14px body, 12px metadata (no 10-11px)
- No horizontal scroll anywhere
- Cards have adequate thumb-interaction padding
- `tel:` links for all phone numbers

## Backend Changes

### None to existing APIs

The ingestion pipeline, Supabase schema, briefing engine, flight query API, and chat endpoint remain unchanged. Only changes:

### Unified query routing (new)

Merge the flight search widget and omnipresent chat into a single component that:
1. Parses user input using existing `parseFlightIntent()`
2. Routes flight intents to `/api/flights/query`
3. Routes general questions to `/api/chat`
4. Renders results inline in a unified format

### New sources in lib/sources.ts

Add entries for flydubai, Air Arabia, Qatar Airways, GCAA, UK FCDO, Australian DFAT, Canadian GAC.
